# 乐库整理助手

面向 Windows 11 的本地音乐文件整理工具。M1～P7 功能已经接入；P8 曾完成一次本机交付，但当前源码在该构建之后又合入了恢复修复，因此最新 `HEAD` 的 EXE、安装目录和桌面快捷方式仍待重新构建与回读验证。

## 当前安全边界

- 开发、自动化测试和独立验收只使用 `TemporaryDirectory` 创建的媒体夹具与数据库，不读取或修改用户真实的 Downloads、Music 或其他媒体目录。
- 正式应用只处理用户显式选择的目录，并仅在用户应用数据目录保存索引。“导入”入口现已接入 P6 安全移动导入：先生成只读预览，用户再次确认后才执行；窗口内另有“只读扫描已有音乐”入口，只建立索引而不移动文件。
- P2-A 只读分析、P2-B 同目录安全重命名及 P2-C 候选副本标签同步均已接入；数据库失败会恢复原标签文件和原文件名。
- P3 只改进正式列表的查询、歌名优先搜索、稳定排序和扫描差异状态，不扩大文件写入范围。
- P4 只接入 `.lrc` 扫描、编码安全读取、候选匹配和关系保存；低置信度结果必须人工确认。
- P5 只管理用户显式选择歌单目录中的 Windows `.lnk`；移除歌单项不会删除音乐文件。
- P2 不删除文件、不跨目录移动、不计算导入去重哈希；WAV、OGG、AAC 只允许安全重命名。
- P5 已支持歌单目录、添加/移除 `.lnk`、重命名后的受管快捷方式联动，以及联动失败后的持久化记录和启动自动重试。当前重命名确认弹窗尚不预先统计受影响快捷方式数量，联动仍在音乐重命名成功后作为独立后台步骤执行。
- P6 已支持经大小与 SHA-256 校验的安全移动导入；P7 已支持备份删除、恢复与到期清理。
- M1 章节和截图保留的是历史 UI 原型证据；当前生产控制器中的 P1～P7 流程不是模拟按钮。

本项目是音乐文件管理工具，不是音乐播放器。

本机安装路径：`%LOCALAPPDATA%\Programs\MusicCtrl\MusicCtrl.exe`。桌面快捷方式名称为“乐库整理助手”。

## 环境要求

- Windows 11
- Python 3.13.5（当前源码与隔离构建环境已验证版本）
- PySide6（版本要求见 `requirements.txt`）
- Mutagen（固定版本见 `requirements.txt`）

## 获取并运行

```powershell
git clone https://github.com/VexorChan/MusicCtrl.git
cd MusicCtrl
pip install -r requirements.txt
python main.py
```

主窗口默认尺寸为 `1200 × 760`，最小尺寸为 `960 × 600`，支持最大化和自由缩放。

当前可用流程分为两类：

- 只读建库：点击“导入” → “只读扫描已有音乐” → 选择音乐目录 → 开始扫描；成功后主列表刷新并记住目录，取消或失败不会覆盖上一次成功目录。
- 安全移动导入：点击“导入” → 选择源目录、目标目录和音频/歌词模式 → “生成只读预览” → 核对重复、冲突和失败项 → “开始安全移动导入” → 在确认框再次确认。只有大小与 SHA-256 校验通过的项目才会原子落位并删除源文件，同名文件不会被覆盖。

## 生成验收截图

```powershell
python capture_screenshots.py
```

脚本会在 `screenshots/` 目录生成完整的 10 张 M1 验收截图：所有音乐、所有歌词、歌单、导入、重命名预览、歌词匹配、操作历史、设置、删除音乐确认和搜索无结果。
