# AGENTS.md

## 适用范围

- 本文件适用于项目根目录及所有子目录。
- 开始任何任务前，必须先阅读：
  - `docs/requirements.md`
  - `docs/implementation-plan.md`
  - `docs/acceptance-criteria.md`
  - 涉及数据库时还必须阅读 `docs/database-design.md`
- M1～P8 已通过功能门禁；当前处于 **本机自用稳定交付维护**。
- P8 必须在独立构建目录生成 Windows 可执行程序，验证应用数据不写入安装目录，并创建可回读的桌面快捷方式。

## 技术栈

- 使用 Python 3.12、PySide6 和 SQLite。
- 中文界面使用 Microsoft YaHei UI，英文和数字可使用 Segoe UI。
- UI 优先使用 PySide6 自带组件和统一 QSS，不引入大型第三方 UI 框架。
- 正式数据表格使用 Qt Model/View；原型中的简单模拟表格可以使用 `QTableWidget`。
- 读写文本文件一律使用 UTF-8。
- 不引入不必要的新依赖。

## 产品边界

- 这是本地音乐文件管理工具，不是音乐播放器。
- 禁止擅自增加播放、进度条、音量、封面、在线搜索、账号、推荐、专辑管理或视频功能。
- 不得增加需求中不存在的导航分类和高风险操作。
- M1 中所有操作只能使用内存模拟数据，不能读取、修改、移动或删除用户的真实音乐及歌词文件。
- P1 产品只读扫描只允许枚举用户在应用中显式选择的根目录；生产数据库路径固定到用户应用数据目录，禁止放在项目根或媒体目录。
- P1 取消被观察后不再开始新批次或发送该批信号；已经进入的单个原子事务允许完成并计入 `cancelled(count)`，随后不得 reconcile 或覆盖上次成功目录。
- P2 产品只处理用户在应用中显式选择且已由 P1 建立索引的音频文件；不得把未选择文件或仅凭路径发现的文件自动纳入分析或执行。
- P2-A 只读读取 Title/Artist；只有两者同时有效才采用标签，否则整体按最后一个半角 `-` 回退解析文件名，生成可编辑预览且不修改媒体；P2-B 仅在用户确认后执行同目录安全重命名，禁止覆盖；P2-C 仅通过候选副本写入并验证受支持格式的元数据，失败时恢复原文件。
- P2 不移动或导入文件，不创建或更新 Windows 快捷方式；快捷方式联动属于 P5，移动导入属于 P6。
- P3 只重构正式列表查询、搜索、排序和既有只读扫描差异同步；不得新增标签写入、跨目录移动、删除、哈希或快捷方式操作。
- P4 只扫描 `.lrc`，读取文本时不得改写编码或内容；内嵌歌词优先，低置信度候选不得自动提交，匹配关系只通过 repository 保存。
- P5 只在用户显式选择的歌单根内创建和管理 `.lnk`；移除歌单项只删除受管快捷方式，绝不删除目标音乐；目标音乐必须来自 active 音频索引。
- P6 采用临时目标、大小与 SHA-256 双重校验和原子落位；校验成功前绝不删除源文件，同名不同内容绝不覆盖。

## 架构规则

- UI 层不得直接访问 SQLite 或执行文件操作。
- 所有数据库变更必须通过 repository 层。
- 文件系统是事实来源；SQLite 只保存索引、关系、设置和审计信息。
- 长时间扫描、哈希、复制和移动必须在工作线程运行，禁止阻塞 Qt 主线程。
- 危险操作必须使用“计划 → 预览 → 确认 → 执行 → 校验 → 提交日志”的流程。
- 文件操作失败时不得把数据库记录标记为成功。
- 部分成功必须保存到文件级操作明细，不能只记录一个批次状态。
- 不得硬编码密钥、Token、密码或数据库连接串。

## 文件操作安全规则

- 禁止在没有确认弹窗的情况下删除真实音频、歌词、快捷方式或备份文件。
- 默认导入方式是移动；只有目标文件完成大小与 SHA-256 校验后，才能删除源文件。
- 同名目标文件不得自动覆盖。
- SHA-256 相同判定为重复；目标路径相同但哈希不同判定为内容冲突。
- 删除音乐或歌词时先移动到备份目录，默认保留 7 天。
- 永久清理备份必须再次确认，并展示即将删除的数量和路径范围。
- 恢复备份时遇到同名文件不得覆盖，必须进入冲突处理。
- 删除歌词前必须检查引用关系。
- 音乐移动或重命名后的受管歌单快捷方式更新留到 P5 实现；P2 不得提前写入快捷方式。
- 测试真实文件操作时只能使用测试创建的临时目录，禁止使用用户的 Downloads、Music 或其他真实媒体目录。
- 不删除项目文件，除非用户明确要求且已经说明依据、影响和恢复方式。

## 文件名与元数据规则

- 统一文件名格式是 `歌名-歌手.ext`，连接符前后不留空格。
- 只有 Title 和 Artist 同时存在且非空时才采用标签；任一缺失时不得拼接半标签结果，必须整体回退到文件名解析。
- 文件名回退解析使用最后一个半角 `-` 分隔歌名和歌手。
- 无法唯一识别时标记为“待手动确认”，不得自动执行。
- Windows 非法字符替换成对应全角字符或空格，并在预览中明确显示。
- 多歌手统一使用 `、`；英文组合名保持原样。
- P2 保留操作前文件名、路径和原始元数据快照，用于审计与失败恢复；哈希与移动导入留到 P6。
- P2 元数据写入只支持 MP3、FLAC、M4A，并且必须先写候选副本、回读验证，再安全替换；WAV、OGG、AAC 只允许重命名文件。
- `missing` 文件禁止进入执行；`external_changed` 文件必须重新分析并由用户再次确认，不能沿用旧预览。
- 检测到内嵌歌词时优先使用内嵌歌词，并标记“已有内嵌歌词”。

## P2 依赖与分发边界

- Mutagen 按 GPL 许可使用时，当前私人、本机、自用开发可以推进。
- P8 生成的 `.exe` 不得上传或分享；如需对外分发，必须先确认许可证兼容性并准备对应源码与构建材料，或者更换为许可兼容的依赖。
- 当前阶段不新增项目 `LICENSE` 文件，许可证决策留到外部分发方案确定时处理。

## 开发工作流

1. 开始编码前，先说明当前阶段、准备修改的文件和验证方式。
2. 每次只实现当前阶段内的一个可运行纵向功能切片。
3. 不得采用“先写完所有数据库、再写所有界面、最后补业务”的机械分层开发方式。
4. 优先小步修改，复用已有目录结构、风格和工具链。
5. 遇到未确定的业务规则时先标记风险并请求确认，不得强行猜测。
6. 完成修改后运行与修改范围最相关的测试和静态检查。
7. 汇报：改了什么、为什么改、如何验证、已知风险。
8. 产生仓库改动的任务包通过定向测试与监督边界复核后，立即原子提交并推送；纯测试、设计或调查任务不创建空提交。
9. 阶段门禁用于关闭当前阶段并进入下一阶段；用户已授权负责人按门禁结果连续推进后续阶段，不重复等待确认。

## 当前测试命令

在项目根目录执行：

```powershell
python -m compileall -q .
python smoke_test.py
```

后续引入 `pytest` 后增加：

```powershell
python -m pytest
```

- 如果测试无法运行，必须说明原因并提供人工验证步骤。
- 不得为了覆盖率编写无意义测试。

## 代码质量

- 类型标注优先，函数职责保持单一。
- 业务状态使用枚举或明确的数据类型，不散落魔法字符串。
- 路径使用 `pathlib.Path` 或专用路径服务处理。
- 数据库事务边界放在应用用例或 repository 协调层，不放在控件事件中。
- 用户可见错误必须给出可理解的信息；详细异常写入操作日志。
- 不自动执行危险命令，不重置生产配置，不清空数据库。
