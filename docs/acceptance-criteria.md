# 验收标准

## 1. 完成定义

一个功能只有同时满足以下条件才算完成：

1. 行为符合 `requirements.md`。
2. 实现范围符合当前 `implementation-plan.md` 阶段。
3. 相关自动化测试通过。
4. 没有触碰阶段禁止的真实资源。
5. 已汇报修改文件、验证方式和已知风险。
6. 产生仓库改动的任务包通过定向测试与监督边界复核后，已经原子提交并推送。
7. 用户完成阶段实际操作验收；该验收控制阶段发布和进入下一阶段，不阻止已通过门禁的任务包提交。

仅仅“窗口能打开”或“代码能运行”不等于功能完成。
纯测试、复核或设计任务没有仓库改动时，不创建空 Git commit。

## 2. M1 UI 原型总体验收

### 2.1 运行

- 能以 Python 源码启动。
- 默认窗口为 1200 × 760，最小尺寸为 960 × 600。
- 支持最大化和自由缩放。
- 最小尺寸下控件不重叠，核心按钮不会被挤出窗口。
- 程序关闭后没有持续运行的模拟任务。

### 2.2 安全边界

- 不扫描真实 Downloads 或 Music 目录。
- 不创建或写入 SQLite 数据库。
- 不计算真实文件哈希。
- 不移动、复制、重命名或删除真实文件。
- 不创建或修改 Windows 快捷方式。
- 所有危险按钮只展示模拟预览或模拟结果。

### 2.3 视觉一致性

- 使用 Windows 11 浅色桌面工具风格。
- 字体、间距、按钮、输入框、表格、状态色和弹窗一致。
- 颜色只用于导航选中、主要操作、焦点、选择和必要状态。
- 不出现播放器、专辑封面、播放控件、网页后台卡片堆叠或手机 App 风格。
- 中文没有明显截断，表格文本和按钮文字可理解。

## 3. 主窗口验收

- 左侧导航固定宽度，主内容区域随窗口扩展。
- 所有音乐、所有歌词和每个歌单均可切换。
- 歌单搜索实时过滤歌单名称。
- 新建歌单入口可打开弹窗。
- 全局按钮顺序严格为导入、重命名、匹配歌词、操作历史、设置。
- 顶部五个按钮均为纯文字，静止状态视觉一致。
- 悬浮时文字约增大 10% 且按钮外框不移动，按下时背景变灰，释放或移出后恢复。
- 顶部工具栏规则不得改变主操作、危险操作、导航和筛选按钮的语义样式。

## 4. 所有音乐验收

- 表格列严格为选择框、歌名、歌手、时长、格式、大小、歌词状态。
- 不出现专辑、年份、路径、比特率、修改时间、所属歌单或封面列。
- 行高约 38～42 px，表头固定。
- 支持单选、多选和表头全选。
- 未选择音乐时，添加到和删除按钮禁用。
- 右键菜单只显示打开所在文件夹、重命名、重新匹配歌词。
- 双击行不执行任何操作。
- 歌词状态包含未检查、已匹配、可能匹配、未匹配、冲突、已有内嵌歌词。

## 5. 搜索与排序验收

自动化测试至少覆盖：

1. 输入首尾空格会被忽略。
2. 英文大小写不影响结果。
3. 有歌名结果时不混入歌手结果。
4. 没有歌名结果时才匹配歌手。
5. 完全一致排在开头匹配和包含匹配之前。
6. 匹配度相同时保持原列表顺序。
7. 无结果显示“没有找到匹配的音乐”。
8. 顶部排序菜单包含需求规定的八种音乐排序方式。
9. 当前排序菜单项有勾选状态。
10. 点击表头可以排序并显示方向。

## 6. 所有歌词验收

- 表格列严格为选择框、歌名、歌手、格式、大小、歌词状态。
- 不显示时长。
- 模拟数据覆盖已匹配、可能匹配、未匹配、冲突和未检查。
- 删除预览明确说明真实版本会移动到备份并检查引用关系。

## 7. 歌单验收

- 表格结构与音乐表格一致。
- 页面明确说明移除快捷方式不会删除音乐。
- 添加到菜单包含歌单搜索、多个可选歌单和新建歌单。
- 已存在项目显示“已存在，将自动跳过”。
- 明确说明不会复制音乐文件。

## 8. 导入窗口验收

- 窗口约 900 × 620，可调整大小。
- 音频和歌词扫描模式互斥。
- 音频模式只表达 MP3、FLAC、WAV、M4A、OGG、AAC。
- 歌词模式只表达 LRC，不出现 TXT 支持。
- 字段名称是“扫描文件夹”和“目标文件夹”。
- 表格列严格为 No.、名称、状态。
- 模拟状态覆盖可导入、名称待确认、重复、冲突、校验、成功和失败。
- 移动按钮状态符合可操作项数量。

## 9. 重命名预览验收

- 表格显示选择框、原文件名、建议文件名、识别来源、状态。
- 建议文件名可以编辑。
- 同步 ID3 默认勾选。
- 明确说明执行前显示完整预览。
- 示例覆盖 ID3、文件名、无法识别和冲突。
- 非法字符替换在预览中可见。
- 无法唯一识别的项目不能自动勾选执行。

## 10. 歌词匹配验收

- 包含未匹配、重复与冲突两个标签页。
- 使用音频列表和候选歌词左右分栏。
- 候选项显示置信度。
- 支持模拟自动匹配、手动选择、取消、批量处理和忽略。
- 内嵌歌词优先标记为“已有内嵌歌词”。
- 不提供内嵌歌词导出按钮。

## 11. 历史、设置和确认弹窗验收

- 历史筛选包含全部、导入、重命名、删除、歌单、歌词匹配。
- 历史明细包含文件、原路径、目标路径、结果、原因和时间。
- 只有最近一次符合条件的完整导入显示撤销入口。
- 设置包含路径、重命名、备份、数据维护。
- 备份默认选中 7 天。
- 重命名前预览强制开启且不可关闭。
- 删除音乐弹窗列出部分文件、备份说明和歌词复选框。
- 永久清理必须有二次确认设计。
- 取消按钮在前，确认按钮在后，所有弹窗保持一致。

## 12. 截图验收

必须提供完整清晰的：

1. 所有音乐主页面
2. 所有歌词页面
3. 歌单页面
4. 导入窗口
5. 重命名预览窗口
6. 歌词匹配窗口
7. 操作历史窗口
8. 设置窗口
9. 删除音乐确认弹窗
10. 搜索无结果状态

截图检查：

- 没有控件溢出和中文截断
- 工具栏不拥挤
- 弹窗按钮层级清晰
- 状态颜色不过度刺眼
- 没有播放器或网页后台特征
- 至少根据截图完成一轮视觉调整

## 13. 当前自动化检查

在项目根目录执行：

```powershell
python -m compileall -q .
python smoke_test.py
```

两条命令均成功后，才能进入人工 UI 验收。若当前 Python 环境没有 PySide6，应明确报告环境缺口，不得声称验证通过。

## 14. 后续真实功能的安全验收门槛

- 所有文件操作测试只使用测试临时目录。
- 导入目标未完成大小和 SHA-256 校验时，源文件必须仍然存在。
- 同名不同哈希不能覆盖。
- 删除后的文件能从备份恢复。
- 恢复遇到同名文件不能覆盖。
- 数据库提交失败时不能把文件操作记录为成功。
- 文件操作失败时必须保留可诊断的操作明细。
- 数据库记录与磁盘不一致时必须标记差异，不能静默删除记录。
- 快捷方式更新失败必须报告，且能通过补偿步骤恢复一致性。
- 永久清理备份必须在操作时再次确认。
